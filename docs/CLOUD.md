# Cloud Deliverable 1

1. Fully implement each service. Each student must be responsible for one service. This doesn’t mean that you cannot work as a team to finish the implementation.
2. Provide documentation for the interface of each service. What methods do they expose? How can you invoke these methods (HTTP method, input and output data, input and output data format)?
3. Prepare unit tests for each service. Make sure that your tests are as complete as possible.
4. Create a docker image that can be used to deploy each service separately.
5. Write a CI/CD pipeline that will use the automated tests and build scripts (i.e., docker image) to automatically integrate changes and deploy the service on a container.
